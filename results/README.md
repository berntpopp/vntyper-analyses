# Results Directory Structure

This directory contains all pseudonymized results generated from joint cohort analyses using VNtyper 2. The structure is designed to facilitate clear organization and easy retrieval of results from different analysis types and contributing centers.

---

## Folder Overview

### 1. `validation/`
- **Purpose:**
  - Store validated results, typically organized by contributing center or cohort.
  - Each center or cohort should have its own subfolder (for example folder Paris for results coming from Imagine Institute in Paris).
- **Contents:**
  - Subfolders named after each center. This could be city name of lab name (e.g., `Paris/`, `Berlin/`, etc.).
  - Each subfolder contains the center's validated result files, such as:
    - Genotyping result tables in standard and fast mode (e.g., `renome_fast_mode.csv`, `renome_slow_mode.csv`) merged with Additional summary statistics generated by extract_additional_stats.py script and merged with csv fiel generated by vntyper cohort analysis.
    - HTML reports (e.g., `renome_fast.html`, `renome_slow.html`)

### 2. `screening/`
- **Purpose:**
  - Hold results from screening cohorts such as GCKD cohort etc.
- **Contents:**
  - Screening result files (HTML, CSV) plus additional statatistics like explain in the validation section.

### 3. `aggregated/`
- **Purpose:**
  - Store combined genotyping results and summary statistics from multiple cohorts or centers.
  - Useful for meta-analyses and joint reporting (visualization of the results).
- **Contents:**
  - Aggregated CSV/TSV files, summary tables, and overall statistics.
  - Data dictionaries describing the fields in summary files.


#### Example Structure:
```
validation/
  Paris/
    renome_fast_mode.csv (this is generated by vntyper cohort on results obtained from fast mode analysis merged with additional statistics generated by extract_additional_stats.py)
    renome_slow_mode.csv (this is generated by vntyper cohort on results obtained from slow mode analysis merged with additional statistics generated by extract_additional_stats.py)
    renome_fast.html (obtained from running vntyper cohort)
    renome_slow.html (obtained from running vntyper cohort)
  London/
    ...
```
#### CSV file must contain information about the capture kits used (Twist, Kapa, Agilent etc) in the library preparation. 

---

## Organizing Results from Different Centers

- **Create a subfolder for each center or cohort** under `validation/` or `screening/`using a clear, consistent naming convention (e.g., `Paris/`, `London/`, `Berlin/`).
- **Place all validated result files for that center** in its respective subfolder. Include both tabular data (CSV/TSV) and any associated reports (HTML, PDF, etc.).
- **For joint or meta-analyses**, we will merge results from different centers and will store it in `aggregated/`
- **For initial or screening analyses**, use the `screening/` folder.
- **Add a README file** to any new subfolder if additional explanation is needed specific to your cohort/center (e.g., details about the cohort, special notes on data processing).

---

## Notes
- All files must be pseudonymized according to project SOPs.
- Avoid storing raw or identifiable data in this directory.
- If you have questions about where to place a file, consult the project coordinators or open issue in this repository.
